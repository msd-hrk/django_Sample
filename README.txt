【開発方針】
・TDD（テスト駆動開発）
	tests.pyにテストを記述するねんけど、views.pyに機能を実装して、tests.pyにテストを書く
	テストを実行して全部通ったら、コミットする流れでいけたらいいなーと思います。
	テスト駆動開発は初めてなんで、チュートリアルに書いてたしせっかくなんでやってみようとおもって（笑）
	ちなみに、テストはgitbashで以下コマンド実行でOK
		python manage.py test polls
	
・開発期間無制限なんで暇なときに手伝ってあげるってスタンスでオッケーです。
・なんなら完成しなくても特に問題ないです。（笑）
・ミッターも誘おうかと思ってます。（ミッターが余裕できてきたら）
